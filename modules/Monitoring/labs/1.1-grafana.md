## Part 1: What is Grafana?

Grafana is an open-source analytics and visualization platform. It allows you to:

- **Visualize data** from many different sources (databases, APIs, CSV files, etc.)
- **Create dashboards** with charts, graphs, and alerts
- **Monitor applications and infrastructure** in real-time
- **Collaborate** with teams on shared dashboards

### Why Grafana for Observability?

| Use Case | Description |
|----------|-------------|
| Application Monitoring | Track response times, error rates, throughput |
| Infrastructure Monitoring | CPU, memory, disk, network metrics |
| Incident Response | Quickly identify and troubleshoot issues |
| Alerting | Get notified when metrics exceed thresholds |

Grafana supports many data sources including Prometheus, InfluxDB, Elasticsearch, MySQL, PostgreSQL, and even simple CSV files!

## Part 2: Running Grafana with Docker

Grafana is an application that runs as a web server. We'll run it using Docker so you can see it in action quickly.

### Step 1: Start Grafana Container

Run the following command to start Grafana:

```bash
docker run -d --name grafana -p 3000:3000 grafana/grafana
```

**What this command does:**
| Flag | Purpose |
|------|---------|
| `-d` | Run container in detached (background) mode |
| `--name grafana` | Give the container a friendly name |
| `-p 3000:3000` | Map port 3000 on your host to port 3000 in the container |
| `grafana/grafana` | The official Grafana Docker image |

### Step 2: Verify the Container is Running

Check that Grafana is running:

```bash
docker ps
```

You should see output similar to:

```
CONTAINER ID   IMAGE             COMMAND     STATUS          PORTS                    NAMES
abc123def456   grafana/grafana   "/run.sh"   Up 10 seconds   0.0.0.0:3000->3000/tcp   grafana
```

### Step 3: View Application Logs

Grafana is a real applicationâ€”it writes logs just like any other service. View them with:

```bash
docker logs grafana
```

You'll see startup messages like:

```
logger=settings t=2026-01-17T12:00:00Z level=info msg="Starting Grafana" version=10.x.x
logger=http.server t=2026-01-17T12:00:01Z level=info msg="HTTP Server Listen" address=0.0.0.0:3000
```

**Pro tip:** To follow logs in real-time (like `tail -f`):

```bash
docker logs -f grafana
```

Press `Ctrl+C` to stop following.

### Step 3: Create a Dashboard with CSV Data

## Part 7: Understanding Grafana for Observability

Grafana shines when used for **application observability**. Here's why teams use it:

### The Three Pillars of Observability

| Pillar | Description | Grafana Support |
|--------|-------------|-----------------|
| **Metrics** | Numeric measurements over time | Prometheus, InfluxDB dashboards |
| **Logs** | Text records of events | Loki integration |
| **Traces** | Request flow through services | Tempo, Jaeger integration |

### How Grafana Improves Reliability

1. **Proactive Monitoring** - See problems before users report them
2. **Faster Incident Response** - Dashboards show what's wrong quickly
3. **Historical Analysis** - Understand trends and patterns
4. **Alerting** - Get notified when things go wrong

### Real-World Example

Imagine your web application slows down:

1. **Without Grafana:** Users complain â†’ You scramble to find the issue â†’ Downtime
2. **With Grafana:** Alert fires â†’ Dashboard shows high CPU â†’ You scale resources â†’ Minimal impact
